# 6️⃣ 캐싱

![image.png](image%202.png)

- 정의: 자주 사용하는 데이터를 임시 저장(캐시)하여, 같은 요청이 반복될 때 빠르게 제공하는 기술
- 목적
  1. 속도 향상 → 데이터베이스나 서버에 다시 요청하지 않고, 캐시에서 빠르게 가져올 수 있음
  2. 서버 부하 감소 → 같은 데이터를 여러 번 요청할 필요가 없어 서버의 부담이 줄어듦
  3. 네트워크 비용 절감 → 불필요한 데이터 전송이 줄어들어 효율적
- 종류
  | 캐싱 종류 | 설명 |
  | ------------------ | ------------------------------------------------------------------------------------------------- |
  | 브라우저 캐시 | 웹 브라우저가 이미지, CSS, JavaScript 등을 저장하여 페이지 로딩 속도를 높임 |
  | CDN 캐시 | 콘텐츠 전송 네트워크(Content Delivery Network)가 데이터를 캐싱하여 전 세계적으로 빠른 서비스 제공 |
  | 서버 캐시 | 서버에서 데이터를 캐싱하여 반복적인 요청에 빠르게 응답 (예: Redis, Memcached) |
  | 데이터 베이스 캐시 | 자주 조회하는 데이터를 DB에 저장하는 대신 메모리에 캐싱 (예: MySQL Query Cache) |
